## 🐛 二、Bug 追蹤與排除紀錄

### Bug 1：Python 連 HTTPS 報 SSL 憑證錯誤
* **🛑 錯誤情境 / 訊息**：`ssl.SSLCertVerificationError: CERTIFICATE_VERIFY_FAILED：unable to get local issuer certificate`（但 `curl` 同一網址卻正常）。
* **🔍 發生原因**：python.org 版的 Python 在 macOS **不使用系統的憑證庫**，自己帶一份卻沒裝好；`curl` 走系統憑證所以沒事。
* **✅ 排除方法**：用 `certifi`（或 `/etc/ssl/cert.pem`）建立 SSL context，所有 `urlopen` 都帶 `context=SSL_CTX`。
* **🛡️ 避坑指南**：macOS 上 Python 連線出現憑證錯誤，第一步先試 `import certifi; certifi.where()`，把它餵給 `ssl.create_default_context(cafile=...)`；別急著關閉憑證驗證（不安全）。

### Bug 2：基本面抓到「過期日期」的數據（PE 顯示 30 而非 31.9）
* **🛑 錯誤情境 / 訊息**：連續打很多次 TWSE API 後，個股本益比/淨值比變成幾天前的舊值。
* **🔍 發生原因**：TWSE 對短時間大量請求會**限流**，回傳非 OK；原本程式遇到就「安靜跳過」，導致最新月份那筆漏掉，退而採用更舊的資料。
* **✅ 排除方法**：寫一個 `twse_json()`：`stat` 非 OK 就**退避重試**（1.2s→2.4s→…），成功後仍稍歇；並把重複抓取（加權指數抓兩次）改成抓一次共用。
* **🛡️ 避坑指南**：抓公開 API 時，「沒資料」和「被限流」要分開處理——限流要重試，不能默默跳過。資料看起來「怪怪的舊舊的」時，先懷疑是不是被限流吃掉了最新一筆。

### Bug 3：Gemini 2.5-pro 無法使用（HTTP 429 quota=0）
* **🛑 錯誤情境 / 訊息**：`429 ... limit: 0, model: gemini-2.5-pro`。
* **🔍 發生原因**：pro 級模型在**免費方案額度為 0**，需綁定帳單才能用。
* **✅ 排除方法**：改用免費可用、推理最強的 **`gemini-3-flash-preview`**，並開啟 thinking。
* **🛡️ 避坑指南**：接新 LLM 前，先用 ListModels／實際打一次確認「這個 key／方案能用哪些模型」，不要憑名字猜。

### Bug 4：Gemini 偶發 503 服務忙線
* **🛑 錯誤情境 / 訊息**：`HTTP Error 503: Service Unavailable`（preview 模型尖峰時）。
* **🔍 發生原因**：preview 模型負載高時會短暫不可用。
* **✅ 排除方法**：對 `429/500/503` 加退避重試（3s→6s→9s）。
* **🛡️ 避坑指南**：對「每天只跑一次的重要任務」，外部 API 一定要有重試；preview 版模型尤其要防忙線。

### Bug 5：波克夏 BRK.B 的 A／B 股數據混淆（位階 -762% 的荒謬結論）
* **🛑 錯誤情境 / 訊息**：BRK.B 現價 $495 卻顯示 52週區間 $685,150~$775,000、EPS $50,396、位階 -762%，且技術面「資料不足」。
* **🔍 發生原因**：兩個雷同時踩到——① **Yahoo 代號格式不同**：Yahoo 特別股用 `BRK-B`（連字號），我傳了 `BRK.B`（點），抓不到歷史 → 技術面空白；② **Finnhub 對 BRK.B 回傳 A 股財務**（EPS、52週是 A 股的天價數字）。
* **✅ 排除方法**：① Yahoo 代號自動 `.` → `-`；② **52週區間改用 Yahoo 歷史自己算**（股別正確）；③ **EPS 改用 現價÷本益比 推算**（PE 是 A/B 通用比率）。
* **🛡️ 避坑指南**：跨資料源整合時，**同一檔股票在不同平台的代號格式可能不同**（`.` vs `-`）；遇到「數字大到不合理」先懷疑是不是抓錯股別/單位。能用「比率」反推的（EPS＝價/本益比）就別硬信單一來源的絕對值。

### Bug 6：Telegram bot 的 log 一直是空的
* **🛑 錯誤情境**：常駐程式在跑，但 `bot.log` 沒任何內容。
* **🔍 發生原因**：Python 輸出到「非終端機」時會**緩衝**，程式不結束就不刷新。
* **✅ 排除方法**：launchd plist 設 `EnvironmentVariables → PYTHONUNBUFFERED=1`。
* **🛡️ 避坑指南**：背景／常駐服務看不到 log，先想到「輸出緩衝」，設 `PYTHONUNBUFFERED=1` 或 `print(..., flush=True)`。

## 💡 三、新手技能提煉（費曼學習法）

* **【MCP 是什麼】**：把它想成 AI 的「萬用轉接頭（USB Hub）」。AI 本身只會聊天，但插上不同的 MCP，它就能「動手」操作不同設備——插上 GitHub MCP 就能讀你的程式碼、插上 Telegram MCP 就能幫你發訊息、插上股市 MCP 就能查股價。今天就是幫 AI 插了一排這種轉接頭。

* **【數據歸數據、AI 歸解讀（分層架構）】**：好比餐廳分工——**「秤重的磅秤」（程式）負責量出精準的數字**，**「主廚」（AI）負責把食材煮成一道菜（解讀）**。我們絕不讓主廚去「猜」食材幾公克（不讓 AI 編造數字），它只能用磅秤量好的數據來發揮。這就是為什麼我們堅持「價格、均線、本益比都由程式算，AI 只負責讀數據給觀點」——數字永遠正確，AI 不會幻覺。

* **【退避重試 Backoff Retry】**：像打電話遇到忙線。笨方法是「掛掉立刻重撥、一直撞」，反而更塞。聰明做法是「等 3 秒再撥，還是忙就等 6 秒、9 秒」。今天對 TWSE 限流、Gemini 503 都用這招——**失敗不要硬撞，越撞越久越退一步**，成功率反而高。
